# source_audits_and_reports

This is the repo governance layer.

Use this directory for documents that describe the state of the source set,
rather than the fictional or design content itself.

## Typical Contents

- source manifests
- migration ledgers
- package changelogs
- thread archive audits
- gap-hunt maps and acquisition checklists
- refresh notes and migration package readmes

## Update This Directory When

- the source inventory changes materially
- a package is retired, reclaimed, or archived
- a compendium refresh changes source coverage assumptions
- recovery work changes what is considered safely preserved

## Working Rule

If a future repo user needs to answer "what exists, what moved, what was
retired, and what is still missing?", the answer should usually live here.
